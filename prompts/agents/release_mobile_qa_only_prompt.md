# ReleaseOrchestrator - Mobile QA Only (No Code Changes)

목표: 메인 개발과 충돌 없이 모바일 회귀 QA 기준을 문서화하고 릴리스 게이트를 준비한다.

## 절대 제약
- 코드 파일(`notedrive/src/**`) 수정 금지
- 기존 구현 로직 변경 금지
- 문서 파일만 수정: `docs/qa_log.md` (필수), 필요 시 `docs/handoff.md`의 QA 섹션만 보강

## 작업 범위
1. `docs/qa_log.md`에 모바일 회귀 테스트 체크리스트 추가
   - 검색: 검색바 토글, 실시간 필터, 검색어 클리어, 결과 없음 메시지
   - 선택: 선택 모드 진입/해제, 체크 인디케이터, 다중 선택 유지
   - 이동: 단건/다건 이동, 폴더 선택 시트, 성공/부분실패 토스트
   - 삭제: 단건/다건 삭제, 확인 다이얼로그, 토스트
   - 데스크탑 보완: 1단 사이드바 숨김 후 재오픈 UI 동작

2. 각 시나리오에 아래 항목을 표 형식으로 기록
   - ID
   - 사전조건
   - 수행 절차
   - 기대 결과
   - 실제 결과(빈칸)
   - 상태(PASS/FAIL/BLOCKED)

3. 릴리스 게이트 문구 추가
   - `P0 실패 0건`, `BLOCKED 0건`일 때만 릴리스 가능

## 검증
- 문서 링크/경로 오탈자 확인
- 체크리스트만 추가됐는지 `git diff -- docs/qa_log.md docs/handoff.md`로 확인

## 결과 보고 형식
1. 추가/수정한 QA 시나리오 수
2. 수정 파일 목록(절대경로)
3. 릴리스 게이트 조건
4. 남은 리스크
