# 프로젝트 컨텍스트 (Project Context)

최종 업데이트: 2026-02-22

## 1) 프로젝트 목표
- UpNote 스타일의 사용성을 가진 Markdown 노트 앱을 구축한다.
- 로컬 파일(Obsidian 호환)과 Google Drive 연동을 모두 지원한다.
- 데스크톱(Mac/Windows) 우선 완성 후 모바일/Always-on 운영으로 확장한다.

## 2) 현재 제품 정의 (현재 범위)
- 제품명: `NoteDrive`
- 핵심 구성:
  - 3-pane 노트 UI (폴더/목록/에디터)
  - 노트/폴더 CRUD, 드래그 이동, 휴지통 복원
  - 해시태그 자동완성/하이라이트
  - 로컬(`local`) 및 Drive API(`gdrive`) 스토리지 모드

## 3) 사용자 시나리오
- 데스크톱에서 빠르게 노트를 작성/정리하고, Markdown 파일을 직접 관리한다.
- Google Drive Desktop 또는 Drive API를 통해 기기 간 동기화한다.
- 실수 삭제 시 휴지통에서 복원한다.

## 4) 핵심 원칙
- 로컬 퍼스트: Markdown 파일을 표준 포맷으로 유지한다.
- 복구 가능성 우선: 삭제는 soft delete(휴지통)로 처리한다.
- 문서/코드 일치: 실행 방식과 문서를 항상 동기화한다.
- 단계적 확장: 모바일 UX -> APK -> Always-on 서버 순서로 확장한다.

## 5) 현재 우선순위
- `P1` 모바일 UX 고도화 (하단 내비게이션, 터치 전환 최적화)
- `P1` Always-on 서버 기반 구축 완료 (인프라 구성 완료, 실 배포 대기)
- `P1` Android APK 전환 준비 (Capacitor)

상세 우선순위는 `docs/tasks.md`를 기준으로 관리한다.

## 6) 비범위 (현재 단계)
- 완전한 모바일 네이티브 앱 배포 완료
- 대규모 멀티유저 협업 기능
- 고도화된 로컬 LLM 자동화 파이프라인 상시 운영

## 7) 성공 기준
- 문서만 보고 신규 환경에서 실행/검증 가능
- 핵심 노트 플로우(생성/수정/이동/삭제/복원) 안정 동작
- 태그/검색/정리 기능이 실사용 수준으로 동작
- 모바일 확장 단계로 자연스럽게 이어질 수 있는 구조 유지
