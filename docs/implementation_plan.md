# 모바일 UX 고도화 Phase 2 구현 계획

최종 업데이트: 2026-02-22

## 목표
- 모바일 실사용 동선 완성: 검색/선택/더보기 액션을 실제 기능과 연결
- 데스크탑 편의성 보강: 사이드바 접힘 상태에서도 즉시 복귀 가능

## 이번 세션 반영 사항 (Codex)
1. 데스크탑 사이드바 복귀 플로팅 트리거 추가
2. 모바일 노트 진입 슬라이드 애니메이션 제거(즉시 전환 중심)
3. 모바일 검색 아이콘 -> 상단 검색바 토글 + 실시간 리스트 필터링 연결
4. 모바일 더보기 메뉴 주요 액션 연결
   - 새 노트북
   - 링크 복사
   - 노트 이동
   - 노트 삭제
   - 정렬 모드 순환(최신순/오래된순/제목순)
   - 선택 모드 토글
   - 설정 진입 유도

## 남은 작업 (Phase 2)
1. 노트 이동 UX 개선
- 현재는 폴더 ID 직접 입력 방식이므로, 폴더 선택 시트(UI)로 교체 필요

2. 선택 모드 시각 보강
- 노트별 체크 인디케이터와 일괄 액션 바(이동/삭제/해제) 추가

3. 모바일 검색 UX 마감
- 검색 결과 없음 상태 문구 개선
- 검색어 클리어 버튼/최근 검색어 도입 검토

4. QA 자동화/회귀 체크 강화
- 모바일/데스크탑 필수 시나리오를 `docs/qa_log.md`와 연결

## 역할별 다음 액션 (실행 단위)
### ProductOwner
- Phase 2 남은 작업의 DoD 확정
- "이동/선택/검색" UX 우선순위 재배치

### FrontendUXEngineer
- 이동 대상 선택 UI(시트) 구현
- 선택 모드 일괄 액션 바 + 선택 상태 시각화 구현
- 모바일 검색 micro-interaction 정리

### PlatformEngineer
- 노트 다중 이동/삭제 액션의 서버 액션 안정화
- 이동/삭제 실패 케이스 토스트/재시도 정책 보강

### SecurityPrivacyEngineer
- 링크 복사/공유 시 노트 식별자 노출 정책 검토
- 모바일 액션 로그에 민감정보 노출 여부 점검

### AIEngineer
- 검색/정렬/선택 흐름에 붙일 AI 보조 기능(요약/태그추천) 진입점 설계

### ReleaseOrchestrator
- Phase 2 완료 기준 기반 검증 체크리스트 운영
- 문서(`tasks/handoff/changelog`) 동기화 및 릴리스 게이트 관리
