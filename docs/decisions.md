# 결정 기록 (Decisions / ADR)

최종 업데이트: 2026-02-21

## ADR-001: Git 저장소를 문서/코드 단일 기준으로 사용
- **상태**: 승인됨
- **결정**: 개발 문서와 코드의 최신 상태는 저장소 기준으로 관리한다.
- **이유**: 세션/환경이 바뀌어도 동일한 기준점에서 복구하기 위함.

## ADR-002: 단일 컨텍스트 문서 체계 유지
- **상태**: 승인됨
- **결정**: 문서 시작점은 `docs/docs_map.md`로 통일하고, 중복 컨텍스트 파일 생성을 지양한다.
- **이유**: 문서 분산으로 인한 충돌과 해석 불일치를 줄이기 위함.

## ADR-003: 심볼릭 링크 기반 운영 금지
- **상태**: 승인됨
- **결정**: 실행/문서 경로에 심볼릭 링크를 사용하지 않는다.
- **이유**: OS별 경로 이슈와 런처 오류를 예방하기 위함.

## ADR-004: 사용자 커뮤니케이션/문서는 한국어 우선
- **상태**: 승인됨
- **결정**: 사용자 안내 문서와 운영 문서는 한국어 중심으로 작성한다.
- **이유**: 실사용자 기준 가독성과 유지보수 속도를 높이기 위함.

## ADR-005: 앱 기본 스토리지 전략은 Local 우선
- **상태**: 승인됨
- **결정**: 기본 모드는 `local`로 유지하고, 필요 시 `gdrive` API 모드를 선택적으로 사용한다.
- **이유**: 로컬 파일(Obsidian 호환) 중심 워크플로우가 구현/운영 복잡도가 낮기 때문.

## ADR-006: 삭제는 휴지통(Soft Delete) 우선
- **상태**: 승인됨
- **결정**: 노트/폴더 삭제 시 즉시 영구삭제하지 않고 휴지통으로 이동한다.
- **이유**: 실사용에서의 오삭제 복원 가능성을 보장하기 위함.

## ADR-007: 태그 UX는 실시간 인식 + 자동완성 채택
- **상태**: 승인됨
- **결정**: `#` 입력 시 해시태그를 즉시 인식하고 자동완성/하이라이트를 제공한다.
- **이유**: 태그 입력 피로를 줄이고 분류 품질을 높이기 위함.

## ADR-008: 모바일 UX 패턴은 UpNote 스타일 채택
- **상태**: 승인됨
- **결정**: 모바일은 FAB + 하단 내비게이션 + 단순한 탐색 흐름을 기준으로 설계한다.
- **이유**: 터치 기반 사용성/가독성이 검증된 패턴을 우선 적용하기 위함.

## ADR-009: macOS 런처는 안정 실행 경로 우선
- **상태**: 승인됨
- **결정**: macOS 런처는 포트 정리 후 `clean build -> start` 경로를 우선 사용한다.
- **이유**: `dev` 실행에서 발생한 `.next` 산출물 불안정 이슈를 줄이기 위함.

## ADR-010: Always-on 요구 시 클라우드 서버 도입
- **상태**: 승인됨 (다음 단계)
- **결정**: PC 상시 구동이 어려운 경우 Cloud Run 기반 운영을 목표로 한다.
- **이유**: 모바일 단독 사용성과 상시 접근성을 확보하기 위함.

## ADR-011: 개발/검수 도구 역할 분리
- **상태**: 승인됨
- **결정**: 본 개발은 Antigravity에서 진행하고, Codex는 변경 검수/정리/문서 동기화 중심으로 운영한다.
- **이유**: 동일 워크트리 동시 편집으로 인한 충돌 위험을 줄이고 작업 책임을 명확히 하기 위함.
