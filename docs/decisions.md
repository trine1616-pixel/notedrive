# 결정 기록 (Decisions)

## ADR-001: GitHub를 단일 진실 공급원으로 사용
- **상태**: 승인됨
- **결정**: 모든 개발 상태와 문서는 GitHub 저장소에 저장되어야 한다.
- **이유**: 다양한 환경에서의 데이터 일관성과 접근성 보장.

## ADR-002: 단일 context 파일 시스템
- **상태**: 승인됨
- **결정**: 모든 LLM에 대해 동일한 `project_context.md`를 사용한다. 모델별 중복 파일을 허용하지 않는다.

## ADR-003: 심볼릭 링크 사용 금지
- **상태**: 승인됨
- **결정**: 컨텍스트 관리에 심볼릭 링크를 사용하지 않는다.
- **이유**: OS 간 이식성 및 안정성 문제 방지.


## ADR-005: 한국어 중심 사고 및 문서화
- **상태**: 승인됨
- **결정**: 모든 사고 과정(Thought Process), 사용자 응답, 프로젝트 문서는 한국어로 작성한다.
- **이유**: 개발자의 가독성을 극대화하고 프로젝트 전체의 언어 일관성을 유지하기 위함.

## ADR-007: 로컬 LLM 모델 선정 (Gemma 3 12B)
- **상태**: 승인됨 (보류 조건부)
- **결정**: 초기 모델로 `gemma3:12b`를 사용하되, 성능 저하 발생 시 `8B` 이하 모델로 전환한다.
- **이유**: 
    - 12B 모델의 VRAM 요구량은 약 8GB로, M1 16GB 시스템에서 OS(4GB) 및 기타 앱(4GB)과 공존하기에 "타이트한" 수준임.
    - 백그라운드 작업 시 시스템이 버벅일 가능성이 있음.
    - 대안 모델: `llama3.1:8b`, `gemma2:9b`, `qwen2.5:7b` (약 4~5GB 소모).
