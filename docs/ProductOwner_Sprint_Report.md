# NoteDrive 스프린트 백로그 및 역할 분담 (by Product Owner)

최종 업데이트: 2026-02-22

## 1. 우선순위 표 (Priority Table)

이번 스프린트의 핵심 목표는 **"모바일 실사용성 확보"**와 **"지속 가능한 클라우드 운영 설계"**입니다.

| 우선순위 | 구분 | 태스크명 | 핵심 목표 |
| :--- | :--- | :--- | :--- |
| **Must (P1)** | UX | **모바일 UX 고도화** | 갤럭시 등 실기기 기준 폴더 -> 목록 -> 에디터 전환 최적화 |
| **Must (P1)** | Platform | **Always-on 아키텍처 설계** | [완료] Cloud Run 배포 및 Docker 최적화 |
| **Must (P1)** | Platform | **Android APK 전환 준비** | Capacitor 기반 패키징 및 권한/동기화 전략 수립 |
| **Must (P1)** | Admin | **운영 문서 일원화** | 실행/설정/검증 절차를 문서만으로 재현 가능하도록 정비 |
| **Should (P2)** | UX | **모바일 키보드 포커스 개선** | 새 노트 생성 시 커서/가상 키보드 자동 활성화 |
| **Should (P2)** | Platform | **이미지/첨부 경로 호환성** | Mac/Win/Android 간 경로 차이 및 렌더링 검증 |
| **Could (P3)** | AI | **LLM(Ollama) 연동 고도화** | 요약/태그 추천 기능의 UX 통합 (모바일 우선순위 뒤로 보류) |

## 2. 기능별 완료 기준 (DoD: Definition of Done)

각 주요 태스크가 완료되었다고 판단하기 위한 구체적인 검증 기준입니다.

### [UX] 모바일 UX 고도화
- [ ] 갤럭시 등 안드로이드 실기기 브라우저에서 3-pane 레이아웃이 겹침 없이 동작한다.
- [ ] 하단 내비게이션(UpNote 스타일)을 통해 폴더/검색/설정 간 이동이 1-depth로 가능하다.
- [ ] 리스트에서 노트를 탭했을 때 에디터로 즉시 전환되며, 뒤로가기 시 리스트로 복귀한다.

- [x] GCP Cloud Run 배포를 위한 `Dockerfile` 및 설정 파일이 포함되어 있다.
- [x] OAuth 2.0 및 Drive API를 통한 파일 동기화 흐름이 설계 문서화되어 있다.
- [x] 프리 티어 범위 내 운영을 위한 비용 가드레일(인스턴스 제한 등)이 설정 가이드에 포함된다.

### [Admin] 운영 문서 일원화
- [ ] 신규 개발자가 `docs/handoff.md`만 보고 로컬 환경 셋업부터 실행까지 10분 내 완료 가능하다.
- [ ] 모든 ADR(결정 사항)이 `docs/decisions.md`에 최신화되어 있다.

## 3. 역할별 작업 할당안

6개 역할 체계에 따른 책임과 세부 작업 할당입니다.

| 역할 (Role) | 담당 태스크 | 세부 내용 |
| :--- | :--- | :--- |
| **ProductOwner** | **백로그 및 우선순위 관리** | `docs/tasks.md` 관리, 사용자 요구사항 분석 및 DoD 확정 |
| **FrontendUXEngineer** | **모바일 UX 및 인터랙션** | 모바일 전환 플로우 구현, 키보드 포커스 개선, UI 가독성 점검 |
| **PlatformEngineer** | **코어 로직 및 클라우드 설계** | Cloud Run 아키텍처 수립, Drive API 연동 안정화, 경로 호환성 해결 |
| **SecurityPrivacyEngineer** | **인증 및 데이터 보안** | OAuth Scope 최소화, 환경변수(Secrets) 관리 전략, 권한 검토 |
| **AIEngineer** | **AI 기능 프로토타이핑** | 요약/분류 기능의 백엔드 연동 준비 (P1/P2 작업 지원) |
| **ReleaseOrchestrator** | **통합 및 검증 관리** | 브랜치 병합 관리, `docs/qa_log.md` 기반 최종 검수, 릴리스 결정 |

---

> [!IMPORTANT]
> **Product Owner의 제언**
> 현재 최우선 순위는 **'모바일에서 실제로 쓸 수 있는 앱'**을 만드는 것입니다. Platform과 Frontend는 모바일 UX 전환 플로우를 최우선으로 협업해 주시기 바랍니다.
